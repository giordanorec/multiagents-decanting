# Example sessions

The plugin is domain-agnostic. Below are example topics that exercise different parts of the pipeline.

## Product / feature

```
/brainstorm standard

# When prompted:
Topic: New features for a personal-finance app aimed at gig workers in Latin America who earn variable income.
```

Expected: morphological axes around `delivery channel`, `income type`, `feedback loop`, `social dimension`. Strong persona diversity (gig drivers, delivery workers, freelance designers, informal vendors).

## Research direction

```
/brainstorm deep

Topic: Promising directions for combating LLM mode collapse in 2027.
```

Expected: axes around `intervention layer` (training / inference / RLHF), `metric of diversity`, `application domain`. Boden transformations likely after round 6 when "training-time" axis saturates.

## Creative / open-ended

```
/brainstorm marathon

Topic: A new kind of public ritual for shared remote workspaces.
```

Expected: very wide divergence in round 1, slow clustering, many minority-voice critiques (different cultural framings). Tier marathon allows the axes to transform mid-run.

## Self-referential test

```
/brainstorm quick

Topic: Ways the brainstorm skill itself could be improved.
```

Useful for sanity-checking the system on its own output.
